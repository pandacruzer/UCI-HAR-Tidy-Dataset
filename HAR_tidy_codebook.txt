Feature 
=======

The features in this database is a processed version of the Human Activity Recognition Using Smartphones Dataset[1]. This dataset combines the test and train data and outputs the average of all mean and standard deviation meansurements by the six activities and test subjects.

The features include:

Time domain signals and acceleration signals (prefix 't' to denote units of time).

tBodyAcc-XYZ
tGravityAcc-XYZ
tBodyAccJerk-XYZ
tBodyGyro-XYZ
tBodyGyroJerk-XYZ
tBodyAccMag
tGravityAccMag
tBodyAccJerkMag
tBodyGyroMag
tBodyGyroJerkMag

These signals were used to estimate variables of the feature vector for each pattern:  
'XYZ' is used to denote 3-axial signals in the X, Y and Z directions.

Similarly, a set of features are created using Fast Fourier Transform (FFT), note the prefix 'f' to indicate frequency domain signals

fBodyAcc-XYZ
fBodyAccJerk-XYZ
fBodyGyro-XYZ
fBodyAccMag
fBodyAccJerkMag
fBodyGyroMag
fBodyGyroJerkMag

mean() variables indicates average of the measurements and std() indicates standard deviation of the measurements.

Lastly, the attribute columns keep track of the activity and test subject in the test

subject: the unique identifer assigned to each volunteer test subject
act_num: the number that indicates the activity being performed
act_label: the name of the activity being performed
